Hiểu về Các Cấu Trúc Dữ Liệu trong Dự Án Node.js

Trang này giải thích các cấu trúc dữ liệu phổ biến và cách chúng có thể được áp dụng trong các dự án thực tế sử dụng Node.js, Express, và TypeScript.

1. Mảng (Array)

Mảng là một cấu trúc dữ liệu lưu trữ một danh sách các phần tử theo thứ tự nhất định. Mảng thường được sử dụng khi bạn cần lưu trữ nhiều mục giống nhau.

Ứng dụng thực tế trong Node.js: Mảng có thể được sử dụng để lưu trữ danh sách người dùng, sản phẩm, đơn hàng, v.v. 

const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' }
];
console.log(users.map(user => user.name)); // ['Alice', 'Bob']

2. Đối Tượng (Object)

Đối tượng là một cấu trúc dữ liệu lưu trữ các cặp khóa-giá trị. Mỗi giá trị có thể được truy cập qua khóa tương ứng.

Ứng dụng thực tế trong Node.js: Đối tượng thường được dùng để lưu trữ thông tin người dùng hoặc cấu hình ứng dụng. 

const user = {
  id: 1,
  name: 'Alice',
  age: 28
};
console.log(user.name); // "Alice"

3. Hash Table (Bảng Băm)

Bảng băm là một cấu trúc dữ liệu ánh xạ giữa khóa và giá trị. Hash table rất hiệu quả trong việc tra cứu nhanh.

Ứng dụng thực tế trong Node.js: Hash Table có thể được sử dụng để lưu trữ thông tin phiên người dùng hoặc dữ liệu cần truy xuất nhanh chóng. 

class HashTable {
  constructor(size) {
    this.table = new Array(size);
  }
  
  hash(key) {
    let hash = 0;
    for (let i = 0; i < key.length; i++) {
      hash += key.charCodeAt(i);
    }
    return hash % this.table.length;
  }
  
  insert(key, value) {
    const index = this.hash(key);
    this.table[index] = value;
  }
  
  search(key) {
    const index = this.hash(key);
    return this.table[index];
  }
}

const hashTable = new HashTable(5);
hashTable.insert('user1', 'Alice');
console.log(hashTable.search('user1')); // "Alice"

4. Ngăn Xếp (Stack)

Ngăn xếp là một cấu trúc dữ liệu theo kiểu LIFO (Last In, First Out), nơi phần tử cuối cùng được thêm vào sẽ là phần tử đầu tiên được lấy ra.

Ứng dụng thực tế trong Node.js: Stack có thể được sử dụng để xử lý các tác vụ đệ quy hoặc các thao tác hoàn tác/redo trong ứng dụng. 

class Stack {
  constructor() {
    this.items = [];
  }

  push(element) {
    this.items.push(element);
  }

  pop() {
    return this.items.pop();
  }
}

const stack = new Stack();
stack.push('task1');
stack.push('task2');
console.log(stack.pop()); // "task2"

5. Hàng Đợi (Queue)

Hàng đợi là một cấu trúc dữ liệu theo kiểu FIFO (First In, First Out), nơi phần tử đầu tiên được thêm vào sẽ là phần tử đầu tiên được lấy ra.

Ứng dụng thực tế trong Node.js: Queue thường được sử dụng trong các hệ thống xử lý yêu cầu (request handling) hoặc các tác vụ cần được xử lý theo thứ tự. 

class Queue {
  constructor() {
    this.items = [];
  }

  enqueue(element) {
    this.items.push(element);
  }

  dequeue() {
    return this.items.shift();
  }
}

const queue = new Queue();
queue.enqueue('task1');
queue.enqueue('task2');
console.log(queue.dequeue()); // "task1"

Các cấu trúc dữ liệu như Array, Object, Hash Table, Stack, và Queue rất quan trọng trong việc phát triển ứng dụng với Node.js và Express. Chúng giúp tổ chức dữ liệu một cách hiệu quả, từ đó cải thiện hiệu suất và khả năng mở rộng của ứng dụng.
